# Phase 2 â€” Fix Copilot inside text threads

## Root cause (confirmed, not speculative)
- `ca23fa7c7c` â€” "copilot: Un-globalify copilot + EditPredictionStore (#46618)" (2026-01-14) is the breaking commit. Before: `Copilot::global(cx)` attached to any editor. After: per-project via `EditPredictionStore::start_copilot_for_project(&project, cx)`, and `crates/zed/src/zed/edit_prediction_registry.rs` `assign_edit_prediction_provider` does `let Some(project) = editor.project().cloned() else { return; }`.
- The restored `text_thread_editor.rs` constructs `Editor::for_buffer(text_thread.read(cx).buffer().clone(), None, window, cx)` â€” `None` project â†’ early return â†’ no provider â†’ Copilot dead.
- History: `82f793144e` added `enabled_in_assistant` setting (renamed `enabled_in_text_threads` at `bdd911f89e`), removed with text threads in `76c6004b27`.

## Steps
1. In restored `text_thread_editor.rs` (`TextThreadEditor::new`, project already a parameter): change `None` â†’ `Some(project.clone())` in `Editor::for_buffer`.
2. Restore `enabled_in_text_threads: bool` (default `true`):
   - `EditPredictionSettings` in `crates/language/src/language_settings.rs` (exact lines in `git show 76c6004b27 -- crates/language/src/language_settings.rs`)
   - matching field in `crates/settings_content/` (grep `copilot: CopilotSettings` to find the struct)
   - default in `assets/settings/default.json` under `"edit_predictions"`
3. Keep restored `editor.set_show_edit_predictions(Some(enabled_in_text_threads))` + `settings_changed` observer.
4. Side-effect checks for passing the project: slash-command completion provider already overrides LSP completions (set in constructor); gutter/code-actions/runnables already disabled via `set_show_*(false)`; registry `observe_new` fires (editor is full-mode with window). If inlay hints appear, disable via current `Editor` API.
5. Fallback if one-line fix has unacceptable side effects â€” manually wire in `TextThreadEditor::new`:
   ```rust

   if let Some(copilot) = ep_store.update(cx, |s, cx| s.start_copilot_for_project(&project, cx)) {
       let provider = cx.new(|_| CopilotEditPredictionDelegate::new(copilot));
       editor.update(cx, |e, cx| e.set_edit_prediction_provider(Some(provider), window, cx));
   ```

   (client/user_store reachable via `project.read(cx).client()`; caveat: hardcodes Copilot, bypasses provider-change observation â€” registry approach preferred.)
